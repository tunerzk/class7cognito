In this lab, I set up a complete authentication and authorization pipeline using Amazon Cognito, API Gateway, and a Lambda Authorizer. You began by creating a Cognito User Pool and adding a user with MFA enabled, then configured an App Client without a client secret so it could be safely used with API Gateway. After enabling the USER_PASSWORD_AUTH flow, you authenticated through the AWS CLI, handled the MFA challenge using a TOTP code, and successfully received an AccessToken, IdToken, and RefreshToken. With the AccessToken in hand, you connected it to your API Gateway setup, where a Lambda Authorizer validated the JWT and returned an IAM policy allowing access. You deployed your API to the prod stage and confirmed that authenticated requests to your /python endpoint returned the expected “Hello from Lambda!” response. This proved that Cognito authentication, JWT validation, and API Gateway authorization were all functioning correctly as a unified system. Next, I plan to explore the version of this flow that uses a client secret + secret hash, which is only appropriate for confidential server‑side clients, not browser or mobile apps.
